# Arena

AI agents compete in games on a live stream. Everything is TypeScript.

## Architecture

See [docs/HIGH_LEVEL_DESIGN.md](docs/HIGH_LEVEL_DESIGN.md) for the full system design.

## Packages

- `poker-api` — rules engine (GraphQL, port 4001)
- `proctor-api` — orchestrator (GraphQL, manages agents + front-end instructions)
- `front-end` — renderer (TTS, animations, no game logic)
- `videographer` — headless browser capture/streaming

## Monorepo Setup

- npm workspaces at `packages/*`
- Root tsconfig: ES2022, Node16, strict
- All packages are ESM (`"type": "module"`)
- Biome for linting/formatting (`npm run check`, `npm run check:fix`), configured globally at root `biome.json`
- `resolverTypes.ts` is generated but checked in (not gitignored) — excluded from Biome

## Common Commands

```sh
npm run poker-api          # dev server with hot reload (tsx watch)
npm run test:poker-api     # vitest
npm run check              # biome lint + format check
npm run check:fix          # biome auto-fix
```

### poker-api specific

```sh
npm run codegen --workspace=@arena/poker-api   # regenerate resolverTypes.ts
npx tsc --noEmit -p packages/poker-api         # type check
```

## Conventions

- GraphQL schema: SDL in TypeScript template literals with `/* GraphQL */` tag, one folder per entity under `src/gql/schema/` (typeDefs.ts + resolvers.ts), merged via `@graphql-tools/merge`
- Resolver types generated by `@graphql-codegen` — use `QueryResolvers["fieldName"]` and `MutationResolvers["fieldName"]` for typing individual resolvers
- Types: prefer re-exporting from generated `resolverTypes.ts` rather than hand-writing duplicates
- Tests: vitest, `_resetGames()` in `beforeEach` for isolation. Integration tests use `yoga.fetch()` via `test/yoga-helper.ts`
- No enum resolvers needed — game-manager returns generated enum values (e.g. `PlayerStatus.Active` = `"ACTIVE"`) which match GraphQL enum names directly
